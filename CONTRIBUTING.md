# Contribution Guidelines

Thank you for your interest in contributing to the Food App Backend! We welcome contributions from developers of all skill levels.

## How to Contribute

### 1. Fork the Repository
- Click the **Fork** button on the repository page
- Clone your fork locally:
  ```bash
  git clone https://github.com/YOUR_USERNAME/food-app-backend.git
  cd food-app-backend
  ```

### 2. Create a Feature Branch
```bash
git checkout -b feature/your-feature-name
```

### 3. Make Your Changes
- Follow the existing code style and structure
- Write clear, descriptive commit messages
- Add tests for new functionality
- Update documentation as needed

### 4. Run Tests
```bash
python -m pytest
python test_business_logic.py
```

### 5. Push Your Changes
```bash
git push origin feature/your-feature-name
```

### 6. Create a Pull Request
- Go to the repository and click **New Pull Request**
- Provide a clear description of your changes
- Reference any related issues

## Code Style

- Follow PEP 8 guidelines
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep functions small and focused

## Reporting Issues

- Check existing issues before creating a new one
- Provide clear, descriptive titles
- Include steps to reproduce the problem
- Attach error messages and logs

## Questions or Discussion

- Open a GitHub Discussion for general questions
- Use Issues for bug reports and feature requests

---

Thank you for contributing! 🎉
